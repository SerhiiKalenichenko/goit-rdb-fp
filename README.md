# goit-rdb-fp

Фінальний проєкт з курсу **Relational Databases**  
Рівень: **MSc / Engineering / Production-minded**

## 1. Мета проєкту

Продемонструвати повний цикл обробки реляційних даних:

- імпорт CSV-даних у MySQL
- нормалізацію таблиці до 3NF
- побудову зв'язків між сутностями
- виконання аналітичних SQL-запитів
- роботу з вбудованими функціями дат
- створення власної SQL-функції

Проєкт виконано на основі датасету `infectious_cases`.

---

## 2. Запуск проєкту

1. Відкрийте MySQL Workbench і підключіться до локального сервера.
2. Імпортуйте CSV-файл `infectious_cases.csv` через **Table Data Import Wizard** у схему `pandemic`.
3. Відкрийте файл `queries.sql` у MySQL Workbench.
4. Виконайте весь файл (`Ctrl+Shift+Enter`) або по блоках (`Ctrl+Enter`).

## 3. Структура рішення

У проєкті реалізовано 5 завдань:

### Task 1. Імпорт даних
- створено схему `pandemic`;
- імпортовано таблицю `infectious_cases`;
- перевірено наявність і кількість завантажених записів.

### Task 2. Нормалізація
- виділено довідник `entities`;
- створено нормалізовану таблицю `infectious_cases_normalized`;
- реалізовано зв’язок `PK/FK`;
- додано індекси для підвищення продуктивності.

### Task 3. Аналітика
Для `number_rabies` виконано розрахунок:
- `AVG`
- `MIN`
- `MAX`
- `SUM`

Результат згруповано за країною (`entity`) та кодом (`code`), відсортовано за `AVG` у спадаючому порядку, виведено топ-10 записів.

### Task 4. Робота з датами
Для кожного року побудовано:
- дату `1 січня` відповідного року;
- поточну дату;
- різницю в роках між цими датами.

### Task 5. Власна функція
Створено SQL-функцію:

`get_year_difference(input_year INT)`

Функція повертає різницю в роках між `1 січня input_year` і поточною датою.

## 4. Архітектура БД

### Вихідна таблиця
`infectious_cases` — сирі імпортовані дані.

### Нормалізовані таблиці

#### entities
Довідник сутностей:
- `entity_id` — первинний ключ;
- `entity` — назва країни/сутності;
- `code` — код країни.

#### infectious_cases_normalized
Фактова таблиця:
- `record_id` — первинний ключ;
- `entity_id` — зовнішній ключ на `entities`;
- `year_value` — рік;
- набір числових показників захворюваності.

---

## 5. Нормалізація

Було виконано нормалізацію до **3NF**.

### 1NF
- усі поля атомарні;
- відсутні повторювані групи.

### 2NF
- дані розділено так, щоб атрибути залежали від ключа повністю;
- довідникову інформацію винесено окремо.

### 3NF
- усунено транзитивні залежності;
- назва сутності та її код більше не дублюються в кожному записі фактів.

У рішенні враховано базові аспекти продуктивності:

- створено індекс за `entity_id`;
