# goit-rdb-fp

Фінальний проєкт з курсу **Relational Databases: Concepts and Techniques**.

## 1. Мета проєкту

У межах проєкту виконано:

- створення окремої схеми `pandemic`
- імпорт набору даних `infectious_cases`
- нормалізацію таблиці до 3NF
- аналітичний запит для `Number_rabies`
- побудову атрибутів дати та різниці в роках
- створення та використання користувацької SQL-функції

## 2. Структура рішення

У проєкті використано дві нормалізовані таблиці:

### `entities`
Довідник сутностей:
- `entity_id` — первинний ключ
- `entity` — назва сутності
- `code` — код сутності

### `infectious_cases_normalized`
Нормалізована таблиця фактів:
- `record_id` — технічний первинний ключ
- `entity_id` — зовнішній ключ на `entities`
- усі числові та часові показники з вихідної таблиці
- колонки `Entity` і `Code` вилучені після нормалізації

## 3. Нормалізація

Початкова таблиця містила дублювання атрибутів `Entity` і `Code`.

Виконана нормалізація до **3NF**:
- **1NF**: атрибути збережено в атомарному вигляді
- **2NF**: неключові поля залежать від ключа запису
- **3NF**: атрибути довідкового типу винесені в окрему таблицю, транзитивні залежності усунено

## 4. Виконані завдання

### Завдання 1
- створено схему `pandemic`
- виконано `USE pandemic`
- перевірено імпорт даних
- виконано `SELECT COUNT(*) FROM infectious_cases`

### Завдання 2
- створено таблицю `entities`
- створено таблицю `infectious_cases_normalized`
- налаштовано `PRIMARY KEY`, `FOREIGN KEY`
- створено індекси для `entity_id` і `Year`

### Завдання 3
Для кожної унікальної сутності пораховано:
- `AVG(Number_rabies)`
- `MIN(Number_rabies)`
- `MAX(Number_rabies)`
- `SUM(Number_rabies)`

Особливості:
- порожні значення відфільтровано
- нечислові значення виключено через `REGEXP`
- результат відсортовано за середнім значенням у спадному порядку
- виведено тільки 10 рядків

### Завдання 4
Для колонки `Year` побудовано:
- дату першого дня року
- поточну дату
- різницю в роках між цими датами

Використані функції:
- `MAKEDATE`
- `CURDATE`
- `TIMESTAMPDIFF`

### Завдання 5
Створено функцію:

`get_year_difference(input_year INT)`

Функція:
- приймає рік
- формує дату `YYYY-01-01`
- повертає різницю в роках між цією датою та поточною датою

## 5. Engineering Notes

У рішенні враховано:
- мінімізацію дублювання даних
- використання `JOIN` замість зайвих підзапитів
- очищення значень перед `CAST`
- індексацію полів, що беруть участь у зв’язках і фільтрації
- відтворюваність SQL-коду

## 6. Security Notes

У проєкті враховано такі елементи безпеки:

- **RBAC / Least Privilege**: права на зміну схеми мають бути лише у довіреного користувача
- **Захист від SQL injection**: використано статичний SQL без dynamic SQL
- **Цілісність даних**: застосовано `PRIMARY KEY`, `FOREIGN KEY`, `UNIQUE`
- **Аудит**: доцільно логувати DDL-операції, імпорт і зміни структури
- **Резервне копіювання**: перед змінами схеми потрібно створювати backup
- **Zero Trust**
- **Secure by Design**
